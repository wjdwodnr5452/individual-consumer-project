**기능 구현**

- **알림서비스**
    - 신청자가 봉사활동 신청 및 취소 시 글쓴이에게 알림 제공
- 세션 클러스터링
    - 봉사 커뮤니티 서버와 알림 서비스 서버 세션 공유를 위한 세션 클러스터링 기능
 
### 상세 내용
- Producer / Consumer 분리 구조 설계
    - 봉사 신청 처리와 알림 전송 로직을 분리하기 위해 Kafka 기반의 Producer–Consumer 아키텍처를 적용했습니다.
    - 봉사 신청과 동시에 Producer 서버에서 이벤트 메시지를 Kafka로 전달하고, Consumer 서버에서 이를 구독하여 후속 처리를 담당하도록 구성했습니다.
    - 이를 통해 신청 처리 로직과 실시간 알림 로직 간의 결합도를 낮추고, 트래픽 증가 시에도 서비스 안정성을 확보할 수 있었습니다.
- Kafka를 활용한 비동기 이벤트 처리
    - 봉사 신청과 알림을 동기적으로 처리할 경우 응답 지연 및 장애 가능성이 있어, Kafka를 통해 비동기 이벤트 방식으로 처리했습니다.
    - 메시지 기반 처리로 인해 알림 서버 장애 시에도 신청 데이터 유실 없이 재처리가 가능하도록 설계했습니다.
- Redis 기반 세션 클러스터링 적용
    - Producer 서버와 Consumer 서버가 다중 인스턴스로 운영되는 환경에서 세션 불일치 문제가 발생하지 않도록 `Redis`를 활용한 세션 클러스터링을 적용했습니다.
- SSE(Server-Sent Events)를 활용한 실시간 알림 구현
    - 봉사 신청 결과를 사용자에게 실시간으로 전달하기 위해 `SSE` 방식을 적용했습니다.
    - WebSocket 대비 구현 복잡도가 낮고, 단방향 알림에 적합하다는 점을 고려하여 SSE를 선택했습니다.
    - 클라이언트는 페이지 새로고침 없이 봉사 신청 상태 변경 및 결과 알림을 수신할 수 있도록 구현했습니다
